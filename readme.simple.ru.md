# Глава 119: Извлечение правил для трейдинга — Простое объяснение

## Что это такое?

Представьте, что у вас есть очень умный друг, который может предсказывать, вырастут или упадут цены на акции. Но когда вы спрашиваете: "Почему ты так думаешь?", он просто отвечает: "Не знаю, просто чувствую". Это же раздражает, правда?

**Извлечение правил** — это как научить этого умного друга объяснять свои мысли простыми правилами "если-то":

- "ЕСЛИ цена падала 3 дня подряд И объём растёт, ТО скорее всего цена вырастет"
- "ЕСЛИ RSI выше 70 И цена достигла сопротивления, ТО скорее всего упадёт"

Теперь вы можете понять, проверить и доверять его предсказаниям.

## Аналогия из жизни

### Проблема шеф-повара

Представьте мастера-повара, который готовит потрясающие блюда. Вы хотите узнать его секреты, но он говорит: "Я добавляю ингредиенты, пока не почувствую, что правильно."

**Извлечение правил** — это как наблюдать за поваром и записывать правила:

```
ЕСЛИ соус слишком густой ТО добавь воды
ЕСЛИ паста ещё твёрдая ТО вари ещё 2 минуты
ЕСЛИ чеснок начинает темнеть ТО сразу убавь огонь
```

Теперь любой может следовать этим правилам и готовить хорошо, даже без "интуиции" повара.

### В терминах трейдинга

Нейронная сеть — как интуитивный повар. Она делает хорошие предсказания, но не может объяснить почему.

Извлечение правил превращает её в кулинарную книгу:

```
ЕСЛИ скользящая_средняя_20 > скользящей_средней_50 ТО рынок бычий
ЕСЛИ волатильность > 2% И всплеск_объёма ТО жди большое движение
ЕСЛИ RSI < 30 И цена_у_поддержки ТО сигнал ПОКУПКА
```

## Зачем это нужно?

### Проблема: Чёрный ящик

Современные модели ИИ (нейронные сети, глубокое обучение) как чёрные ящики:

```
[Вход: Данные о ценах] → [??? Магия ???] → [Выход: Купить/Продать]
```

Мы не видим, что происходит внутри. Это создаёт проблемы:

1. **Доверие**: Вы бы поставили деньги на то, чего не понимаете?
2. **Отладка**: Если модель ошибается, как её исправить?
3. **Регуляторы**: Банки обязаны объяснять свои торговые решения
4. **Обучение**: Нельзя улучшить то, чего не понимаешь

### Решение: Извлечь правила

Извлечение правил открывает чёрный ящик:

```
[Вход: Данные о ценах] → [Понятные правила] → [Выход: Купить/Продать]
                               ↓
                   "ЕСЛИ цена > SMA_20 И RSI < 70
                    ТО ПОКУПАЙ с уверенностью 65%"
```

Теперь вы можете:
- Понимать, почему принимаются решения
- Находить и исправлять проблемы
- Объяснять регуляторам
- Узнавать, какие паттерны важны

## Как это работает?

### Метод 1: Наблюдай и учись (Педагогический)

Мы рассматриваем модель ИИ как учителя и учимся на её примерах:

```
Шаг 1: Показываем ИИ 1000 рыночных ситуаций
Шаг 2: Записываем, что ИИ предсказывает для каждой
Шаг 3: Ищем закономерности: когда говорит "ПОКУПАЙ" vs "ПРОДАВАЙ"
Шаг 4: Записываем эти закономерности как простые правила

Пример:
- ИИ сказал "ПОКУПАЙ" когда цена была выше SMA в 80% случаев
- ИИ сказал "ПРОДАВАЙ" когда RSI был выше 80 в 75% случаев
- Значит: Правило 1: ЕСЛИ цена > SMA ТО склоняйся к ПОКУПКЕ
          Правило 2: ЕСЛИ RSI > 80 ТО склоняйся к ПРОДАЖЕ
```

### Метод 2: Загляни внутрь (Декомпозиционный)

Мы смотрим на внутреннюю структуру ИИ (веса и связи):

```
Внутри нейронной сети:
   Вход: [RSI, Объём, Изменение цены]
              ↓
   Нейрон 1: 0.8×RSI + 0.3×Объём - 0.5×Изменение > 0.5?
              ↓
   Упрощённое правило: ЕСЛИ RSI высокий И Объём средний ТО ...
```

Это точнее, но требует понимания архитектуры ИИ.

### Метод 3: Комбинируй оба (Эклектический)

Используй внутреннюю структуру как подсказку, потом проверяй на примерах.

## Простой пример

### Исходная точка: Обученная модель

Допустим, у нас есть нейронная сеть, предсказывающая направление цены Bitcoin:

```
Вход: [RSI, MACD, Изменение объёма, Волатильность, Отношение к SMA]
Выход: ВВЕРХ (1) или ВНИЗ (0)
Точность: 58% (довольно хорошо для рынков!)
```

Но это чёрный ящик. Нам нужны правила.

### Шаг 1: Генерируем примеры

```python
# Подаём много сценариев в модель
scenarios = generate_market_scenarios(1000)
predictions = model.predict(scenarios)

# Результаты выглядят так:
# Сценарий 1: RSI=45, MACD=0.5, ... → Модель говорит: ВВЕРХ
# Сценарий 2: RSI=75, MACD=-0.2, ... → Модель говорит: ВНИЗ
# ...
```

### Шаг 2: Ищем закономерности

```python
# Используем дерево решений для поиска паттернов
tree = DecisionTree(max_depth=5)
tree.fit(scenarios, predictions)

# Дерево обнаруживает:
# - Когда RSI > 65, модель обычно говорит ВНИЗ
# - Когда MACD > 0.3 И RSI < 50, модель обычно говорит ВВЕРХ
# - Когда Волатильность > 0.03, модель не уверена
```

### Шаг 3: Извлекаем правила

```
Правило 1: ЕСЛИ RSI > 65 ТО ПРОДАВАЙ (уверенность: 72%)
Правило 2: ЕСЛИ MACD > 0.3 И RSI < 50 ТО ПОКУПАЙ (уверенность: 68%)
Правило 3: ЕСЛИ Волатильность > 0.03 ТО ДЕРЖИ (не уверен)
Правило 4: ЕСЛИ Отношение_SMA > 1.02 И Изменение_объёма > 0 ТО ПОКУПАЙ (уверенность: 61%)
```

### Шаг 4: Торгуем по правилам

```
Текущий рынок: RSI=42, MACD=0.45, Вол=0.02, Отношение_SMA=1.03, Изм_объёма=0.1

Проверяем правила:
- Правило 1: RSI=42 не > 65 → не применяется
- Правило 2: MACD=0.45 > 0.3 И RSI=42 < 50 → ПРИМЕНЯЕТСЯ! → ПОКУПАЙ
- Правило 3: Вол=0.02 не > 0.03 → не применяется
- Правило 4: Отношение_SMA=1.03 > 1.02 И Изм_объёма=0.1 > 0 → ПРИМЕНЯЕТСЯ! → ПОКУПАЙ

Решение: ПОКУПАЙ (2 правила согласны)
Объяснение: "Покупаю, потому что MACD положительный при низком RSI,
             и цена выше скользящей средней с растущим объёмом"
```

## Ключевые понятия

| Термин | Простое значение |
|--------|------------------|
| **Чёрный ящик** | Модель, где не видно, как она принимает решения |
| **Извлечение правил** | Получение простых правил если-то из сложной модели |
| **Fidelity (Точность соответствия)** | Насколько точно правила повторяют поведение модели |
| **Coverage (Покрытие)** | Какой процент ситуаций правила могут обработать |
| **Педагогический метод** | Учимся правилам, наблюдая за входами и выходами модели |
| **Декомпозиционный метод** | Учимся правилам, заглядывая внутрь модели |

## Оценка качества правил

### Fidelity: Соответствуют ли правила модели?

```
Предсказания модели:  [КУПИТЬ, ПРОДАТЬ, КУПИТЬ, КУПИТЬ, ПРОДАТЬ, КУПИТЬ]
Предсказания правил:  [КУПИТЬ, ПРОДАТЬ, КУПИТЬ, КУПИТЬ, КУПИТЬ,  КУПИТЬ]
                                                        ↑
                                                  Несовпадение!

Fidelity = 5/6 = 83% (правила совпадают с моделью в 83% случаев)
```

Высокий fidelity означает, что правила точно представляют то, что выучила модель.

### Coverage: Обрабатывают ли правила все случаи?

```
Всего сценариев: 100
Сценариев, где применяется хотя бы одно правило: 92

Покрытие = 92% (правила могут обработать 92% ситуаций)
```

Высокое покрытие означает меньше ситуаций "не знаю".

### Complexity: Достаточно ли правила просты?

```
Простое правило: ЕСЛИ RSI > 70 ТО ПРОДАВАЙ
Сложное правило: ЕСЛИ RSI > 68.5 И MACD < -0.234 И Объём > 1.5×Среднего
                 И Волатильность МЕЖДУ 0.015 И 0.028 ТО ПРОДАВАЙ
```

Более простые правила легче понять и меньше вероятность переобучения.

## Торговая стратегия на правилах

### Генерация сигналов

```
Для каждой новой точки данных рынка:
    1. Рассчитай признаки (RSI, MACD и т.д.)
    2. Проверь каждое правило
    3. Посчитай, сколько правил говорят ПОКУПАЙ vs ПРОДАВАЙ
    4. Сгенерируй сигнал по большинству

Пример:
    Признаки: RSI=35, MACD=0.2, Всплеск_объёма=True

    Правило 1 (ПОКУПАЙ): RSI < 40 → Совпадает! (+1 ПОКУПАЙ)
    Правило 2 (ПРОДАВАЙ): MACD < 0 → Не совпадает
    Правило 3 (ПОКУПАЙ): Всплеск_объёма = True → Совпадает! (+1 ПОКУПАЙ)

    Счёт: ПОКУПАЙ=2, ПРОДАВАЙ=0 → Сигнал: ПОКУПАЙ
```

### Почему это работает

1. **Объяснимость**: "Я купил, потому что RSI в зоне перепроданности и был всплеск объёма"
2. **Настраиваемость**: Не доверяешь Правилу 3? Просто убери его
3. **Отладка**: Стратегия теряет деньги? Проверь, какие правила не работают
4. **Соответствие регуляторам**: Можно показать, почему была совершена каждая сделка

## Сравнение подходов

| Подход | Объяснимость | Точность | Скорость | Лучше всего для |
|--------|--------------|----------|----------|-----------------|
| Нейронная сеть (чёрный ящик) | Никакая | Высокая | Средняя | Когда важна только точность |
| Дерево решений (простое) | Отличная | Средняя | Быстрая | Быстрые, интерпретируемые модели |
| **Извлечение правил** | **Отличная** | **Высокая** | **Быстрая** | **Лучшее из обоих миров** |

## Итог

Извлечение правил — как иметь переводчика, который может объяснить мысли гения простыми словами:

1. **Обучи мощную, но непрозрачную модель** (нейронную сеть)
2. **Извлеки простые правила**, которые приближают её поведение
3. **Торгуй по правилам** с полной объяснимостью
4. **Отлаживай и улучшай**, изучая какие правила работают

Результат: Вы получаете предсказательную силу сложного ИИ с прозрачностью простых правил.
